# Plan por Checkpoints

## Checkpoint 0

- Organizar equipo y flujo de trabajo.
- Acordar estructura documental y estrategia de IA.
- Identificar decisiones de arquitectura de alta criticidad.

## Checkpoint 1

- Diseñar arquitectura base y repositorios.
- Tener CI/CD mínimo en componentes principales.
- Tener al menos un CRUD funcional.
- Presentar prototipos de flujos principales.
- Tener primer set de endpoints OpenAPI del dominio inicial (Usuarios/Perfil).
- Tener al menos 1 diagrama C4 y 1 diagrama de secuencia validados por equipo.
- Definir y documentar estrategia de media (foto de perfil) con alternativa gratis/low-cost y opción local reproducible.

## Checkpoint 2

- E2E funcional con todos los CRUD básicos.
- Avance objetivo de historias obligatorias y optativas acordado con corrector.
- Contratos OpenAPI cubriendo todos los endpoints del alcance implementado.
- Diagramas de arquitectura actualizados respecto de lo implementado.

## Checkpoint 3

- Cerrar historias obligatorias.
- Consolidar optativas según alcance acordado.
- Endurecer calidad, resiliencia y documentación final.
- Cerrar consistencia final entre: enunciado, trazabilidad, ADRs, OpenAPI y diagramas.

## Seguimiento semanal

- Revisar avances contra matriz de trazabilidad.
- Cerrar o replanificar decisiones abiertas.
- Validar coherencia entre ADR, OpenAPI y diagramas.
